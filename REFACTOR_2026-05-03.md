# ListextEditor 重构修复说明 (2026-05-03)

## 修复的问题

### 问题1: 文本框区域拖动触发块移动 ✅ 已彻底修复

**根本原因分析**:
- 原代码使用dragSource变量，逻辑复杂且容易出错
- 事件监听器绑定在dragSource上，但检查逻辑不够清晰
- textarea的事件阻止不够彻底

**重构方案**:
1. **简化拖动源逻辑**：
   - 有textarea的块：只有header可拖动
   - 无textarea的块：整个块可拖动
   
2. **完全隔离textarea**：
   - textarea设置draggable=false
   - 添加mousedown事件阻止冒泡
   - 添加dragstart事件完全阻止
   - 设置cursor=text明确交互意图

3. **提取拖动处理方法**：
   - handleDragStart(e, block)：统一处理拖动开始
   - handleDragEnd(block)：统一处理拖动结束
   - 代码更清晰，易于维护

**修改位置**: src/block-renderer.js attachBlockEvents方法 (第279-403行)

### 问题2: 无法从repeat块中拖出元素 ✅ 已彻底修复

**根本原因分析**:
- 原代码只检查targetBlock是否是repeat，但没有考虑拖动位置
- 任何时候拖到repeat上都会尝试拖出，导致逻辑混乱

**重构方案**:
1. **增加位置检测**：
   - 计算repeat块的header高度
   - 只有当鼠标在header区域时才允许拖出
   - 其他区域返回null，不允许拖动

2. **明确拖出条件**：
   - 必须是repeat块
   - 必须有父容器
   - 必须在header区域

3. **改进用户体验**：
   - 用户需要明确将块拖到repeat的header上方才能拖出
   - 避免误操作
   - 配合"移出重复块"按钮使用

**修改位置**: src/block-renderer.js getDropPlacementForBlock方法 (第382-437行)

## 关键代码变更

### 变更1: 重构attachBlockEvents方法

```javascript
// 重构拖动逻辑：只有header可以拖动
const headerEl = block.querySelector('.block-header');

if (textarea) {
  // 有textarea的块：只允许header拖动
  block.setAttribute('draggable', 'false');
  if (headerEl) {
    headerEl.setAttribute('draggable', 'true');
  }
  textarea.setAttribute('draggable', 'false');
  textarea.style.userSelect = 'text';
  textarea.style.webkitUserSelect = 'text';
  textarea.style.cursor = 'text';
  textarea.addEventListener('input', () => this.onBlockChange());

  // 完全禁用textarea的拖动能力
  textarea.addEventListener('mousedown', (e) => {
    e.stopPropagation();
  });
  textarea.addEventListener('dragstart', (e) => {
    e.preventDefault();
    e.stopPropagation();
  });

  // 只在header上监听dragstart
  if (headerEl) {
    headerEl.addEventListener('dragstart', (e) => {
      this.handleDragStart(e, block);
    });
    headerEl.addEventListener('dragend', () => {
      this.handleDragEnd(block);
    });
  }
} else {
  // 没有textarea的块：整个块可以拖动
  block.setAttribute('draggable', 'true');
  block.addEventListener('dragstart', (e) => {
    this.handleDragStart(e, block);
  });
  block.addEventListener('dragend', () => {
    this.handleDragEnd(block);
  });
}
```

### 变更2: 新增拖动处理方法

```javascript
handleDragStart(e, block) {
  this.draggingBlock = block;
  e.dataTransfer.effectAllowed = 'move';
  e.dataTransfer.setData('text/plain', block.dataset.id);
  block.classList.add('dragging');
  if (!this.placeholderEl) this.placeholderEl = this.createPlaceholder();
}

handleDragEnd(block) {
  block.classList.remove('dragging');
  this.clearPlaceholder();
  this.draggingBlock = null;
  this.restoreAllRepeatEmptyStates();
}
```

### 变更3: 重构getDropPlacementForBlock方法

```javascript
// 重构：处理从repeat块内拖出的情况
if (targetBlock.contains(this.draggingBlock)) {
  // 正在拖动的块在目标块内部
  if (targetBlock.dataset.tagName === 'repeat') {
    // 目标是repeat块，允许拖到repeat外部
    const repeatParent = targetBlock.parentElement;
    if (!repeatParent) return null;

    // 检查是否拖到repeat的header区域（上半部分）
    const repeatRect = targetBlock.getBoundingClientRect();
    const headerEl = targetBlock.querySelector('.block-header');
    const headerHeight = headerEl ? headerEl.getBoundingClientRect().height : 40;

    // 如果鼠标在header区域，允许拖出到repeat外部
    if (clientY < repeatRect.top + headerHeight) {
      container = repeatParent;
      referenceBlock = targetBlock;
    } else {
      // 否则不允许拖动（避免在repeat内部混乱）
      return null;
    }
  } else {
    // 目标不是repeat块，不允许拖动
    return null;
  }
}
```

## 重构优势

1. **代码更清晰**：
   - 拖动逻辑分离到独立方法
   - 条件判断更明确
   - 易于理解和维护

2. **更可靠**：
   - 完全隔离textarea的拖动
   - 明确的拖出条件
   - 减少边界情况

3. **更好的用户体验**：
   - 文本选择不会被打断
   - 拖出操作更明确
   - 减少误操作

## 技术要点

- 使用stopPropagation阻止事件冒泡
- 使用preventDefault阻止默认行为
- 使用getBoundingClientRect精确计算位置
- 分离关注点，提取公共方法
- 明确的条件判断，避免复杂的嵌套逻辑
